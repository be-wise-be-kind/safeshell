SafeShell Architecture

Daemon-based command interception and execution

1. Overview

Key Design Principle

SafeShell uses a warm daemon process to evaluate and execute commands. The shim is a lightweight bash socket client with no Python dependency, enabling minimal overhead.

Socket connection: ~0.5ms Daemon evaluation: ~0.5ms Fork + exec: ~0.5ms Total overhead: ~1.5ms per command

4. Protocol

Execute Request

The execute request type combines evaluation and execution:

{
    "type": "execute",
    "command": "ls -la",
    "working_dir": "/home/user/project",
    "env": {"PATH": "...", "HOME": "..."},
    "execution_context": "human"
}

Response Format

Response includes execution results:

{
    "success": true,
    "decision": "allow",
    "executed": true,
    "exit_code": 0,
    "stdout": "file1.txt\nfile2.txt\n",
    "stderr": "",
    "execution_time_ms": 12
}

For denied commands:

{
    "success": true,
    "decision": "deny",
    "executed": false,
    "denial_message": "Cannot commit directly to protected branch"
}

Protocol models.py RequestType.EXECUTE and response fields for stdout/stderr/exit_code

Manager daemon/manager.py _handle_execute() evaluates then forks to run command
Bash Client shims/safeshell-check Socket client for daemon communication (no Python)
Shim shims/safeshell-shim Command interceptor using bash client
Shell Init shims/init.bash Builtin overrides (cd, source, eval)

6. Design Decisions

  1. TTY handling: Non-interactive only. Buffered stdout/stderr. Interactive commands fall back to direct execution.
  2. Environment inheritance: Child receives env from shim request, merged with daemon env (PATH preserved).
  3. Working directory: Child uses subprocess with cwd parameter to run in requested working_dir.
  4. Output handling: Buffered output returned at end (simpler, sufficient for most commands).
  5. Timeout handling: No timeout - daemon is non-invasive and does not change command behavior.
  6. Builtin overrides: Configurable via check_cd, check_source, check_eval settings. Eval disabled by default due to shell hook overhead.

7. Failure Modes

Daemon Unreachable

Behavior: Fail-open by default. If daemon is unreachable, shim falls back to direct execution (configurable via unreachable_behavior).

Large Output

Behavior: Output is buffered and returned via socket. No artificial limits imposed.

Signal Handling

Behavior: Child process runs in its own process group. Parent daemon handles SIGCHLD. Ctrl+C in terminal sends to child's process group.
